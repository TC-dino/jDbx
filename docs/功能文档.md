## 1. 连接管理

### 1.1 创建连接

**操作步骤：**

1. 点击连接面板底部的「+ 新建连接」按钮
2. 在弹出的对话框中选择数据库类型
3. 填写连接信息
4. 点击「测试连接」验证配置
5. 点击「保存」完成创建

**连接参数说明：**

| 参数   | 必填 | 说明                      |
|------|----|-------------------------|
| 连接名称 | 是  | 自定义名称，方便识别              |
| 主机地址 | 是  | IP 地址或域名                |
| 端口   | 否  | 默认端口会自动填充               |
| 数据库  | 否  | 可以连接后再选择                |
| 用户名  | 是  | 数据库用户名                  |
| 密码   | 是  | 数据库密码                   |
| 颜色标记 | 否  | 用于区分不同环境（支持rgba风格的颜色输入） |

**使用场景：**

```
场景1：本地开发环境
- 名称：Local MySQL
- 主机：localhost
- 端口：3306
- 颜色：绿色

场景2：生产环境（只读）
- 名称：Production MySQL (READONLY)
- 主机：prod-db.example.com
- 端口：3306
- 颜色：红色
- 附加选项：只读模式
```

### 1.2 管理连接

**操作入口：** 点击连接面板底部的「⚙ 连接管理」

**功能列表：**

| 功能   | 说明          |
|------|-------------|
| 编辑连接 | 修改连接配置      |
| 复制连接 | 基于现有连接创建副本  |
| 删除连接 | 删除连接（需确认）   |
| 导出连接 | 导出为 JSON 文件 |
| 导入连接 | 从 JSON 文件导入 |
| 排序连接 | 拖拽调整顺序      |

### 1.3 连接状态

**状态指示：**

| 状态   | 图标 | 颜色 | 说明   |
|------|----|----|------|
| 已连接  | ●  | 绿色 | 连接正常 |
| 未连接  | ○  | 灰色 | 未激活  |
| 连接中  | ◐  | 黄色 | 正在连接 |
| 连接失败 | ✕  | 红色 | 连接异常 |

**自动重连：**

- 断开后自动尝试重连（最多 3 次）
- 重连间隔：1s, 2s, 4s
- 可在设置中关闭

### 1.4 快速连接

**操作步骤：**
我们可以筛选不同的中间件类型（当前支持的），选择后显示对应已保存的连接信息，然后可以根据搜索名字进一步过滤。

1. 在连接面板顶部搜索框输入关键字
2. 实时过滤匹配的连接
3. 回车或双击快速连接

**使用场景：**

```
输入 "prod" → 显示所有包含 "prod" 的连接
筛选 "mysql" → 显示所有 MySQL 连接
筛选 "redis" → 显示所有 Redis 连接
```

注意这里的输入和筛选，是不同的行为。

**补全内容：**

| 类型  | 来源  | 说明      |

**使用示例：**

```sql

SELECT u. FROM users u

-- 输入 sel 自动补全为 SELECT
sel
-- → SELECT

-- 输入 cou 自动补全为 COUNT
SELECT cou(*)
-- → SELECT COUNT(*)
```

### 2.6 SQL 格式化

**操作步骤：**

1. 选中要格式化的 SQL（可选），如果不选择则默认当前SQL编辑器的所有语句
2. 按 Ctrl+Alt+L
3. SQL 自动格式化

或者可以使用邮件上下文菜单的方式触发SQL格式化。

**格式化规则：**

```sql
-- 格式化前
SELECT u.id,u.name,u.email,o.id,o.total FROM users u JOIN orders o ON u.id=o.user_id WHERE u.status='active' AND o.created_at>'2024-01-01' ORDER BY o.created_at DESC LIMIT 10

-- 格式化后
SELECT
    u.id,
    u.name,
    u.email,
    o.id,
    o.total
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE
    u.status = 'active'
    AND o.created_at > '2024-01-01'
ORDER BY o.created_at DESC
LIMIT 10
```

## 3. 查询执行

### 3.1 执行方式

| 方式     | 快捷键              | 说明                |
|--------|------------------|-------------------|
| 执行当前语句 | Ctrl+Enter       | 光标所在语句，自动分析定位当前语句 |
| 执行选中语句 | Ctrl+Shift+Enter | 选中的 SQL           |
| 执行所有语句 | Ctrl+Shift+A     | 整个编辑器内容           |
| 执行到光标处 | Ctrl+Shift+C     | 从开头到光标位置          |

### 3.2 语句识别

**自动识别多语句：**

```sql
-- 编辑器中的内容
SELECT * FROM users;
SELECT * FROM orders;
SELECT * FROM products;

-- 光标在第二条语句上，按 Ctrl+Enter
-- 只执行: SELECT * FROM orders;
```

**语句分隔符：**

- 分号 `;`
- 空行（可选配置）
- GO 关键词（SQL Server 风格）

### 3.3 执行状态

**状态栏显示：**

```
执行中... │ SELECT * FROM users │ ⏱ 0.23s │ 1000 行
```

| 状态   | 说明                                   |
|------|--------------------------------------|
| 执行中  | 显示执行动画                               |
| 执行完成 | 显示耗时和行数                              |
| 执行失败 | 显示错误信息（后续根据错误原因即当前上下文信息，让AI给出更合理的建议） |

### 3.4 查询历史

**查看历史：**

1. 点击结果区域的「历史」标签页
2. 或按 Ctrl+H 打开历史面板

**历史记录内容：**

| 字段   | 说明                       |
|------|--------------------------|
| 时间   | 执行时间                     |
| SQL  | 执行的语句                    |
| 耗时   | 执行耗时                     |
| 状态   | 成功/失败                    |
| 影响行数 | INSERT/UPDATE/DELETE 的行数 |

**操作：**

| 操作      | 说明     |
|---------|--------|
| 双击历史记录  | 加载到编辑器 |
| 右键 → 复制 | 复制 SQL |
| 右键 → 删除 | 删除记录   |
| 清空历史    | 删除所有历史 |

### 3.5 查询超时

**默认设置：**

| 参数   | 默认值 | 说明         |
|------|-----|------------|
| 查询超时 | 30s | 单次查询最大执行时间 |
| 连接超时 | 10s | 连接最大等待时间   |

**修改方式：**

```json
// config.json
{
  "query": {
    "timeout": 30,
    "maxRows": 10000
  }
}
```
