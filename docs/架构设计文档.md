# jDbx 架构设计文档

## 1. 项目概述

### 1.1 产品定位

jDbx 是一款面向**新时代全栈开发者**的轻量级中间件管理工具。

**目标用户画像：**
- 全栈开发者（Frontend + Backend + DevOps）
- 懂 DBA、后端、运维的多面手
- 追求效率，厌恶臃肿
- 个人使用为主，需要高度定制

**与竞品的差异：**

| 维度 | Navicat/DBeaver | jDbx |
|------|-----------------|------|
| 定位 | 通用数据库管理 | 开发者个人效率工具 |
| 功能 | 大而全 | 精而简 |
| 扩展性 | 有限插件 | 完全插件化 |
| 价格 | 付费 | 开源免费 |
| UI 风格 | 传统 | 现代化（Glassmorphism） |

### 1.2 设计原则

1. **插件化优先** - 核心只保留最小功能集，所有中间件支持通过插件实现
2. **开发者体验** - 快捷键驱动，减少鼠标操作
3. **现代化 UI** - 参考 Beekeeper Studio，使用现代设计语言
4. **高性能** - 响应式架构，不阻塞 UI 线程
5. **可扩展** - 每个功能点都可以被插件扩展或替换

## 3. 插件系统设计

### 3.1 插件接口

```java
package dino.jdbx.core.api;

/**
 * 插件接口 - 所有插件必须实现
 */
public interface Plugin {
    
    /**
     * 插件 ID
     */
    String getId();
    
    /**
     * 插件名称
     */
    String getName();
    
    /**
     * 插件版本
     */
    String getVersion();
    
    /**
     * 插件描述
     */
    String getDescription();
    
    /**
     * 插件作者
     */
    String getAuthor();
    
    /**
     * 依赖的其他插件 ID
     */
    default String[] getDependencies() {
        return new String[0];
    }
    
    /**
     * 初始化插件
     */
    void initialize(PluginContext context);
    
    /**
     * 卸载插件
     */
    default void unload() {}
}
```

### 3.3 连接工厂接口

```java
package dino.jdbx.core.api;

/**
 * 连接工厂接口
 * 负责创建和管理数据库连接
 */
public interface ConnectionFactory {
    
    /**
     * 创建连接
     */
    Connection createConnection(ConnectionConfig config) throws Exception;
    
    /**
     * 测试连接
     */
    default boolean testConnection(ConnectionConfig config) {
        try (Connection conn = createConnection(config)) {
            return conn != null && conn.isValid();
        } catch (Exception e) {
            return false;
        }
    }
}
```

### 3.5 插件加载机制

使用 **Java SPI (Service Provider Interface)** 加载插件：

```
META-INF/services/
└── dino.jdbx.core.api.DatabasePlugin
```

文件内容示例：
```
dino.jdbx.plugin.mysql.MySQLPlugin
dino.jdbx.plugin.postgresql.PostgreSQLPlugin
```

**插件加载流程：**

```
1. 启动时扫描所有 jar 包
2. 查找 META-INF/services/dino.jdbx.core.api.DatabasePlugin
3. 通过 ServiceLoader 加载插件类
4. 调用 plugin.initialize(context) 初始化
5. 注册到 PluginRegistry
```

## 8. 错误处理

### 8.1 错误分类

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 连接错误 | 无法连接数据库 | 显示错误提示，建议检查配置 |
| 查询错误 | SQL 语法错误 | 在编辑器中标记错误位置 |
| 执行错误 | 运行时异常 | 显示错误详情和堆栈 |
| 插件错误 | 插件加载/执行失败 | 记录日志，禁用插件 |

### 8.2 错误处理策略

```java
// 查询执行的错误处理
try {
    QueryResult result = connection.executeQuery(sql);
    showResult(result);
} catch (SQLException e) {
    if (isSyntaxError(e)) {
        highlightErrorPosition(e.getLineNumber(), e.getColumnNumber());
    }
    showErrorDialog("查询执行失败", e.getMessage());
    logError("Query execution failed", e);
}
```
